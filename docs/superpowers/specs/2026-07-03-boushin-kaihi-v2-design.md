# 放銃回避トレーニング機能 拡張(v2) 設計

## Context

放銃回避トレーニング3モード（危険牌クイズ・安全度ランキング・押し引き二択）を実装済みの状態で、麻雀ドメインに詳しい3体のサブエージェントに独立レビューを依頼した。3体全員が一致して指摘した2点、および3体中2体が一致して指摘した3点を今回のスコープとして拡張する。ダマテン・鳴き手警戒（2体中1体のみ指摘、かつ「非リーチ他家」という新概念が必要で影響範囲が大きい）は次回に分離する。

### 今回のスコープ

1. **①押し引きへの打点統合**（3体一致・最重要）: 自分の手の価値・相手の想定打点を押し引き判断に反映する。
2. **②複数リーチ時の共通安全牌の教育**（3体一致）: 解説にリーチ者ごとの階級内訳を表示し、「全員に対して安全な牌を探す」発想を可視化する。
3. **③片筋と壁のtier分離**（2体一致）: 現状同一tierに丸められている片筋・壁を分離する。
4. **④筋は両面待ち限定という前提の明示**（2体一致）: シャンポン・単騎・カンチャンには筋・壁・ノーチャンスが通用しないことをUIで注記する。
5. **⑥リーチの巡目情報**（2体一致）: リーチ宣言巡目を表示し、早い巡目のリーチは危険度判定にも反映する。

## データモデル拡張

- `drawSafetyScenario`の各opponentオブジェクトに`isDealer`（真偽値、25%の確率で親と判定）を追加する。既存の`riichiDiscardIndex`をそのまま巡目情報として流用し、巡目 = `riichiDiscardIndex + 1`として扱う。
- `doraTileFromIndicator(doraIndicator)`を新設。数牌はv+1（9の次は1に巡回）、字牌は風牌（東→南→西→北→東）・三元牌（白→發→中→白）でそれぞれ巡回してドラ牌のindexを返す。

## 打点見積もりエンジン

- `estimateOwnValueTier(hand13, doraTile)` — 候補牌を切った後の13枚から簡易役判定（タンヤオ・ピンフ・役牌(三元牌の刻子のみ)・トイトイ・一盃口・混一色）とドラ枚数（`hand13`中の`doraTile`の枚数）を合算して翻数を概算する。正確な符計算は行わない。0〜2翻=LOW、3〜4翻=MID、5翻以上=HIGHの3段階に分類して返す。面子分解は既存`stdShanten`の内部分解ロジックに相当する簡易版を新設し、テンパイ／完成形でなくても「現時点で最も翻数が高くなる分解」を返す。
- `estimateOpponentValueTier(opponent, unseenDoraCount)` — スコア = `(opponent.isDealerなら+1) + (unseenDoraCountが3以上なら+2、1〜2なら+1、0なら+0)`。スコア>=3ならHIGH、2ならMID、0〜1ならLOWの3段階に分類する。
- `unseenDoraCount`はシナリオ全体で共有する値で、`4 - visibleCounts[doraTile]`（0未満にはならない）として算出する。特定の相手の手牌を読むものではなく、「場に見えていないドラ枚数が多いほど、誰かが持っている可能性が上がる」という簡易近似であることをコード注釈に明記する。

## Tier再編(6段階→7段階)

```js
const DANGER_TIERS = {
  GENBUTSU: 0,
  NOCHANCE: 1,
  SUJI_BOTH: 2,
  SUJI_ONE: 3,   // 片筋 — 現物という確定情報に基づくため壁より信頼できる
  KABE: 4,       // 壁 — 搭子構成牌の片方が3枚見え(ノーチャンス一歩手前)、確率的な安全度
  MODERATE: 5,   // 字牌視認1〜2枚・無地端寄り
  DANGEROUS: 6,  // 字牌生牌・無地中央456
};
const TIER_LABELS = ['現物','ノーチャンス','両筋・中筋','片筋','壁','中危険','高危険'];
```

`classifyAgainstOpponent`内の判定ロジック自体（現物→ノーチャンス→両筋/片筋→壁→無地/字牌視認枚数の判定順序）は変更せず、既存の`isKabe`該当ケースをtier3からtier4に、既存のMODERATE/DANGEROUS該当ケースをtier4/5からtier5/6にそれぞれ振り直すのみ。

## 巡目による危険度補正

`classifyAgainstOpponent`が返す基本tierに対し、`overallDangerTier`（または呼び出し元）で以下を適用する:

- 対象opponentの巡目（`riichiDiscardIndex + 1`）が6巡目以内、かつ基本tierがGENBUTSU/NOCHANCEでない場合、tierを1段階危険側に補正する（例: SUJI_ONE(3)→KABE(4)）。
- 補正後の上限はMODERATE(5)までとし、DANGEROUS(6)には引き上げない（巡目だけを理由に最高危険度まで押し上げるのは過剰なため）。
- GENBUTSU・NOCHANCEは巡目に関わらず補正しない（構造的に安全なため）。

## 押し引きロジック刷新

`evaluatePushFold(hand14, candidateTile, scenario)`を以下の順序に刷新する:

1. リーチ0人 → 常に押す（変更なし）。
2. 全リーチ者に対する総合tier（巡目補正後）が GENBUTSU または NOCHANCE（tier<=1）→ シャンテン数に関わらず常に押す（絶対安全牌のため、新規追加）。
3. テンパイ（自分のシャンテン<=0）→ 常に押す（変更なし）。
4. 2シャンテン以上 → 常に降りる（変更なし。ステップ2の絶対安全判定が先に効くため、実質的な挙動は「安全牌以外は降りる」になる）。
5. 1シャンテンのみ、打点を反映した閾値判定を行う:
   - `ownAdj` = `estimateOwnValueTier`がLOW→0、MID→+1、HIGH→+2
   - `oppAdj` = 全リーチ者中の`estimateOpponentValueTier`の最悪値がLOW/MID→0、HIGH→+1
   - `しきい値 = clamp(2 + ownAdj - oppAdj, 2, 5)`（下限はSUJI_BOTH、上限はMODERATE）
   - 総合tier（巡目補正後）<= しきい値 なら押す、それ以外は降りる

## 解説UIの拡張

- 危険牌クイズ・押し引き二択の解説テーブルに、リーチ者ごとの階級内訳列（例:「対リーチ1: 現物」「対リーチ2: 両筋」）を追加する。リーチ1人のみの場合も同じ形式で1列表示する（分岐を増やさないため）。
- `renderOpponents`のリーチ者ラベル表示に巡目（例:「リーチ1(6巡目)」）を追加する。
- 危険牌クイズ・安全度ランキング・押し引き二択の各`modeCaption`の下に、固定の注記文「※筋・壁・ノーチャンスは主に両面待ちが対象です。シャンポン・単騎・カンチャン待ちには通用しません。」を追加する。

## 既存コードへの影響

- `DANGER_TIERS`/`TIER_LABELS`の数値・要素数が変わるため、既存テストのtier数値を参照している箇所（`assertEqual`内の期待値）を新しい数値体系に合わせて更新する。
- `drawSafetyScenario`の戻り値に`isDealer`フィールドが追加されるが、既存のテストフィクスチャ（`{ discards: [...] }`のみのopponentオブジェクト）は`isDealer`が`undefined`のままでも動作するよう、`estimateOpponentValueTier`は`opponent.isDealer`を`Boolean(...)`で扱う。
- `evaluatePushFold`のシグネチャは変更しない（`hand14, candidateTile, scenario`のまま）。

## テスト方針(`test.html`経由で追記)

- `doraTileFromIndicator`: 数牌の巡回(9→1)、風牌の巡回(北→東)、三元牌の巡回(中→白)をそれぞれ検証。
- `estimateOwnValueTier`: タンヤオのみ(LOW)、タンヤオ+ドラ3(HIGH相当)、役なし(LOW)の境界を検証。
- `estimateOpponentValueTier`: 親+ドラ0(スコア1=LOW)、子+未見ドラ3枚(スコア2=MID)、親+未見ドラ3枚(スコア3=HIGH)の境界を検証。
- Tier再編: `isKabe`該当ケースがtier4(KABE)、字牌視認1枚がtier5(MODERATE)になることを検証。
- 巡目補正: 6巡目以内のSUJI_ONE(3)がKABE(4)に補正されること、7巡目以降は補正されないこと、GENBUTSUは巡目に関わらず補正されないこと、MODERATE(5)を超えて補正されないことを検証。
- `evaluatePushFold`: 現物・ノーチャンスなら2シャンテンでも押す(新規)、1シャンテンで自分HIGH+相手LOWならKABE(4)でも押す、1シャンテンで自分LOW+相手HIGHならSUJI_BOTH(2)は押すがSUJI_ONE(3)は降りる、という境界値を検証。

## レビューで指摘され、意図的に許容した設計判断

- ダマテン・鳴き手への警戒は今回のスコープ外（次回に分離）。
- 符計算・厳密な役判定（ペンチャン待ちの否認、七対子の判定、国士無双、三色同順・一気通貫など）は実装しない。翻数の概算にとどめる。
- 相手の打点見積もりは「親か・場のドラ未見枚数」のみの簡易近似であり、相手が実際にドラを持っているかどうかを読む機能ではない。
- 巡目補正は6巡目以内かどうかの単純な二値判定とし、巡目に応じた段階的な補正（例: 3巡目はさらに危険、等）は行わない。
